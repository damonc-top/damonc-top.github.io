    翻译工具链与命令行

    1. 什么是IL2CPP？

    我们提到的IL2CPP技术有两个组成部分：

    • AOT（ahead-of-time）编译器
    • 支持虚拟机的运行时库（runtime library）

    AOT编译器将中间语言Intermediate Language（IL，.NET编译器输出的低阶代码）转换为C++源代码。

    运行时库提供服务和抽象，例如垃圾回收器、对线程和文件的独立于平台的访问，以及内部调用的实现（直接修改托管数据结构的本机代码）。

    1.1. AOT

    IL2CPP的AOT编译器程序是il2cpp.exe。该程序可在Windows平台找到安装的Unity目录Editor /Data /il2cpp，或在OSX找到Unity安装目录Contents /Frameworks /il2cpp /build。

    il2cpp.exe可执行程序完全由C#实现的，在开发IL2CPP期间，我们使用 .NET和Mono编译器对其进行编译。il2cpp.exe程序接受使用Unity附带的Mono编译器编译的托管程序集，并生成C++代码，然后将这些代码传递给特定于平台的C++编译器。

    1.2. runtime

    IL2CPP技术的另一部分是支持虚拟机的运行时库。我们几乎完全使用C++代码实现了这个库（也有一些特定于平台的汇编代码，这两种代码的使用会根据不同的平台保留）。我们将运行时库称为libil2cpp，它作为链接到可执行文件的静态库。IL2CPP技术的主要优势之一是这种简单且可移植的运行时库。

    通过查看安装Unity附带的libil2cpp头文件，您可以找到一些关于libil2cpp代码组织方式的线索（您可以在Windows上的Editor /Data /PlaybackEngines /webglsupport /BuildTools /Libraries /libil2cpp /include目录中找到它们，或者在OSX上位于Contents/Frameworks /il2cpp /libil2cpp目录中）。例如，il2cpp.exe生成的C++代码与libil2cpp运行时之间的接口位于codegen/il2cpp-codegen.h头文件中。

    runtime其中的一个关键部分是垃圾回收器。我们将发布带有libgc（Boehm-Demers-Weiser垃圾回收器的Unity5。但是libil2cpp被设计为允许我们使用其他垃圾回收器。例如，我们正在研究Microsoft GC的集成，它是作为CoreCLR开源的一部分。我们将在本系列后面的关于garbage collector集成的文章中对此进行更多介绍。

    3. IL2CPP不能做什么？

    我们没有试图用IL2CPP重写C#标准库，当您使用IL2CPP构建Unity项目的脚本时，mscorlib.dll、System.dll等所有C#标准库代码与Mono脚本构建所使用的代码完全相同。

    我们依赖于C#标准库代码，这些代码已经为用户所熟知，并在Unity项目中经过了良好的测试。因此，当我们调查一个与IL2CPP相关的bug时，我们可以相当确信这个bug在AOT编译器或运行时库中，而不是其他地方。

    4. 如何开发、测试和交付IL2CPP？

    自2015年1月份IL2CPP 4.6.1p5版本首次公开发布以来，我们已经发布了6个完整版本和7个补丁版本（跨Unity 4.6和5.0版本）。我们已经纠正了发行说明中提到的100多个错误。

    为了实现这种持续的改进，我们在内部只针对一个版本的IL2CPP代码进行开发，该代码位于Unity中用于发布alpha和beta版本的主干分支的边缘。就在每个版本发布之前，我们将IL2CPP更改移植到特定的发布分支，运行我们的测试，并验证我们修复的所有错误都在该版本中得到了纠正。我们的QA和持续工程团队做了令人难以置信的工作，使交付速度成为可能。这意味着我们的用户距离IL2CPP错误的最新修复永远不会超过一周。

    我们的用户社区通过提交许多高质量的错误报告证明了其价值。我们感谢所有来自用户的反馈，以帮助不断改进IL2CPP，我们期待着更多的反馈。

    IL2CPP的开发团队有很强的测试优先心态。我们经常采用测试驱动设计实践，很少在没有好的测试的情况下合并pull request。这种策略对于像IL2CPP这样的技术非常有效，我们有明确的输入和输出。这意味着我们看到的绝大多数bug都不是意外的行为，而是意外的情况（例如，可以使用64位IntPtr作为32位数组索引，导致clang失败并出现C++编译器错误，而真实的代码实际上就是这样做的！）。这种差异使我们能够以高度的信心快速修复错误。

    在我们社区的帮助下，我们正在努力使IL2CPP尽可能稳定和快速。
